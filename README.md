# EasyPaper

Multi-agent academic paper generation service: turns metadata (title, idea, method, data, experiments, references) into structured LaTeX and optionally compiles to PDF.

## Features

- Multi-agent pipeline: planning, writing, review, typesetting, optional VLM layout review
- FastAPI service with health and agent discovery endpoints
- CLI and HTTP API for metadata-driven generation
- LaTeX output with citation validation, figure/table injection, review loop

## Requirements

- Python 3.11+
- LaTeX (pdflatex + bibtex) for PDF compilation
- LLM/VLM API keys — **configure in your own YAML; do not commit real keys**

## Deployment

### 1. Install

```bash
# Recommended: use a virtual environment
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e .
```

### 2. Configure (no keys in repo)

- Copy `configs/example.yaml` to a **private** config file (e.g. `configs/my_config.yaml`).
- Replace every `your_llm_api_key` / `your_vlm_api_key` with your real API keys.

Alternatively, if you use [AgentSociety2](https://github.com/tsinghua-fib-lab/agentsociety): configure EasyPaper LLM/VLM in its config page; it will generate `easypaper_agentsociety.yaml` in the workspace. Point `AGENT_CONFIG_PATH` to that file when starting EasyPaper.

### 3. Start the service

**Foreground (default port 8004 for AgentSociety2):**

```bash
export AGENT_CONFIG_PATH=/path/to/your_config.yaml
./start.sh
# Or: EASYPAPER_PORT=8004 uvicorn src.main:app --host 0.0.0.0 --port 8004
```

### 4. Check

```bash
curl http://localhost:8004/healthz
curl http://localhost:8004/list_agents
```

## Config

Loaded from **`AGENT_CONFIG_PATH`** (default: `./configs/dev.yaml`). Each agent has:

- `model_name` — LLM or VLM model id
- `api_key` — API key (keep secret)
- `base_url` — API base URL

The repo’s `configs/example.yaml` is a **template** with placeholders only. Use your own file with real keys and set `AGENT_CONFIG_PATH` to it.

## Generate a paper

**HTTP API (server must be running):**

```bash
curl -X POST http://localhost:8004/metadata/generate \
  -H "Content-Type: application/json" \
  -d @economist_example/metadata.json
```

**CLI (starts a request to the running server):**

```bash
AGENT_CONFIG_PATH=/path/to/your_config.yaml python scripts/generate_paper.py --input economist_example/metadata.json
```

## Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /healthz` | Health check |
| `GET /config` | Current app config (no secrets in response) |
| `GET /list_agents` | Registered agents and routes |
| `POST /metadata/generate` | Generate paper from metadata JSON |

Agent routes are under `/agent/*` and `/metadata/*`.

## Integration with AgentSociety2

When used with [AgentSociety2](https://github.com/tsinghua-fib-lab/agentsociety):

1. Deploy EasyPaper (this repo) and configure it with your own YAML (or use the file generated by AgentSociety2’s config page).
2. In AgentSociety2 set `EASYPAPER_API_URL` (e.g. `http://localhost:8004` or `http://<host>:8004`).
3. The `generate_paper` tool will call EasyPaper’s `POST /metadata/generate` to produce PDFs from analysis reports.
